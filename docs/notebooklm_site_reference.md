# VPE Prompt Builder 2026 — Полное руководство пользователя v4.0

## Что такое VPE Prompt Builder?

**VPE Prompt Builder** — это профессиональный визуальный конструктор промптов (запросов) для нейросетей. **Сайт НЕ генерирует изображения сам по себе.** Его задача — помочь вам составить максимально точный, детализированный текстовый или JSON-запрос, который вы затем скопируете в нейросеть на вашем выборе (Midjourney, Stable Diffusion, DALL-E 3, Flux и др.).

**Принцип работы:** Вы нажимаете кнопки → сайт собирает профессиональный промпт → вы копируете результат в нейросеть.

---

## Часть 1. Структура интерфейса

Интерфейс состоит из четырёх областей:

```
┌─────────────────────────────────────────────────────────┐
│                    ШАПКА (HEADER)                       │
├──────────────┬──────────────────────────────────────────┤
│              │                                          │
│  НАВИГАТОР   │       ЛЕВАЯ ПАНЕЛЬ                       │
│  КОНСТРУКТОР │      (Все разделы настройки)             │
│  (боковой    │                                          │
│   слайдер)   ├──────────────────────────────────────────┤
│              │       ПРАВАЯ ПАНЕЛЬ                      │
│              │  (Промпт + JSON + Статистика)            │
└──────────────┴──────────────────────────────────────────┘
```

## Часть 2. Шапка (Header)

Шапка зафиксирована в верхней части экрана и не прокручивается.

### Что находится в шапке (слева направо):

| Элемент | Что делает |
|---|---|
| **КОНСТРУКТОР ▶** | Открывает/закрывает боковой слайдер "Навигатора" — панель управления видимостью разделов |
| **Логотип VPE** | Логотип приложения (версия 4.0) |
| **VISUAL PROMPT ENGINEERING DEVELOPER SUITE** | Бейдж темы "Developer Suite". Показывает, какая тема активна |
| **Строка движков** | `MIDJOURNEY / DALL-E 3 / STABLE DIFFUSION / FLUX / IDEOGRAM / GEMINI IMAGEN / CHATGPT IMAGE / NANO BANANA PRO` — отображает поддерживаемые движки |
| **↺ (Отменить)** | Откатывает последнее действие (Undo). Иконка — стрелка назад |
| **Свернуть** | Сворачивает или разворачивает сразу **все** секции в левой панели |
| **Сброс** | Полностью очищает все настройки, текст и выборы — чистый лист |

## Часть 3. Навигатор (Constructor Sidebar / Боковая панель)

Открывается кнопкой **КОНСТРУКТОР ▶** в шапке. Это скрытая панель управления, которая **выезжает поверх основного контента** (overlay overlay). На мобильных устройствах находится в обычном потоке.

### Назначение:
Навигатор позволяет выбирать, **какие разделы будут видны** в левой панели. Ненужные разделы можно скрыть.

### Элементы Навигатора:

| Элемент | Что делает |
|---|---|
| **Отметить все** (чекбокс-переключатель) | Включает или выключает сразу все разделы |
| **Фан** (кнопка) | Быстрый режим: оставляет только самые нужные секции для быстрого выбора без ручных настроек |
| **Pro** (кнопка) | Профессиональный режим: показывает ВСЕ доступные секции для полного контроля |
| **Список секций с переключателями** | Каждая строка — это одна секция левой панели. Тумблер справа включает/выключает её видимость |

## Часть 4. Левая панель — Все разделы подробно

Левая панель — это главная рабочая зона. Все секции по умолчанию **свёрнуты** (закрыты). Кликните по заголовку секции, чтобы развернуть её.

> Пометки `single` и `multi` у заголовков секций обозначают, сколько кнопок можно выбрать: **single** = только одну, **multi** = несколько одновременно.

### 1. 🎬 Режим генерации
**Тип:** multi-toggle (несколько чекбоксов)

Специальные режимы, которые меняют структуру самого запроса. **Эти режимы — не просто теги.** Они полностью пересобирают JSON-запрос по другому шаблону.

| Элемент | Что делает |
|---|---|
| **📽 4 Вариации (Gen 4)** *(тумблер)* | Генерирует 4 кадра одного персонажа с принудительно разными ракурсами: снизу, ковбойский план, голландский угол, 3/4 сзади. Идеально для плейстов и раскадровок |
| **📐 Контакт-лист 3×3** *(тумблер)* | Создаёт сетку 9 кадров (контактный лист) с нарастающей крупностью — от общего плана до детали. Сайт сам формирует 9-панельный JSON со специфической логикой каждого кадра |
| **🔒 Макс. консистентность** *(тумблер)* | Режим жёсткой фиксации внешности персонажа. Нейросеть получает инструкцию не изменять черты лица и форму. Рекомендуется с загруженными референсами |

> ⚠️ Режимы взаимоисключающие между собой и с некоторыми другими функциями (Motion blur нельзя использовать с Gen 4 или 3×3).

### 9. 🤖 AI Модель
**Тип:** single

**Самый важный шаг.** Выбор движка перестраивает весь интерфейс: прячет несовместимые функции, показывает нужные и меняет формат итогового промпта.

| Движок | Особенности |
|---|---|
| **Midjourney** | Итоговый промпт включает параметры `--ar`, `--v`, `--s`, `--c`, `--weird` |
| **DALL-E 3** | Не поддерживает негативный промпт; выдаёт `vivid/natural` и `hd/standard` |
| **Stable Diffusion / Flux** | Поддерживает негативный промпт; веса токенов `(слово:1.5)` |
| **Ideogram** | Хорошо рендерит текст внутри изображения |
| **Nano Banana Pro (NBP)** | Получает только JSON, не текст. Активирует специальные режимы Gen 4 и 3×3 |
| **Gemini Imagen** | Поддержка Google-движка генерации |
| **ChatGPT Image** | Поддержка GPT-4o генерации |

### 19. ⚙️ Параметры съёмки
**Тип:** 4 подраздела, каждый single

Четыре независимых блока управления камерой:

**Диафрагма (Aperture):**
- f/1.2 — максимальная открытость: сильнейшее боке, размытый фон
- f/1.8 — классическое портретное боке
- f/2.8 — кинематографичное боке
- f/4 — баланс чёткости и размытия
- f/8 — стандартная репортажная чёткость
- f/11, f/16, f/22 — максимальная глубина резкости, всё в фокусе

**Фокусное расстояние (Focal Length):**
- 12–14mm — ультраширокий угол, перспективные искажения
- 24mm — широкий угол, архитектура
- 35mm — классический репортаж (видение близкое к человеческому)
- 50mm — нормальный объектив (как видит глаз)
- 85mm — портретный, мягкое сжатие перспективы
- 135mm — телепортрет, сильное боке
- 200mm+ — телефото, сжатие пространства

**Ракурс / Угол (Angle):**
- Eye Level — уровень глаз (нейтральный, человечный)
- Low Angle — снизу вверх (делает объект величественным)
- High Angle — сверху вниз (взгляд с высоты, уязвимость)
- Bird's Eye / Top-Down — птичий полёт, съёмка ровно сверху
- Dutch Angle — наклон камеры (тревога, дезориентация)
- Worm's Eye — взгляд снизу, с уровня пола

**Крупность плана (Shot Size):**
- Extreme Close-Up (ECU) — крупнейший план: глаза, губы, деталь
- Close-Up (CU) — крупный план: лицо целиком
- Medium Close-Up (MCU) — над грудью
- Medium Shot (MS) — по пояс
- Cowboy Shot — по бедро, ковбойский план
- Medium Full Shot (MFS) — ниже колен
- Full Shot (FS) — фигура в полный рост
- Long Shot (LS) — человек и окружение
- Extreme Long Shot (ELS) — общий план, пейзаж
- Overhead Flat Lay — вид сверху (раскладка)
- Силуэт — контр-жур, таинственный силуэт

### 27. ⚙️ Параметры генерации
**Тип:** зависит от выбранного движка (панель меняется)

Технические параметры, специфичные для каждой нейросети.

**Midjourney:** 
- Версия: V7, V6.1, V5.2
- Стиль: Default / Raw (Raw отключает типичное «приукрашивание» MJ)
- Ползунок **Stylize (--s):** 0–1000 (художественность), по умолчанию 250
- Ползунок **Chaos (--chaos):** 0–100 (случайность), по умолчанию 0
- Ползунок **Weird (--weird):** 0–3000 (странность), по умолчанию 0
- Поле **Style Reference (--sref):** URL или ID для ссылки на стиль

**Stable Diffusion:**
- Ползунок **CFG Scale:** 1–20 (1 = свобода интерпретации, 20 = строго по тексту)
- Ползунок **Steps:** 10–50 (больше шагов = лучше качество, но медленнее)
- Подсказка о синтаксе весов токенов: `(важный элемент:1.5)`

**Flux:**
- Модель: Dev (качество), Schnell (скорость), Pro
- Ползунок **Guidance Scale:** 1–10 (аналог CFG)
- Ползунок **Steps:** 4–50

**DALL-E 3:**
- Стиль: Vivid (яркий, насыщенный) / Natural (натуральный, реалистичный)
- Качество: HD / Standard

**Другие движки (без параметров):**
Показывается сообщение: «Все настройки передаются через текст промпта».

### 30. 🚫 Негативный промпт
**Тип:** текстовое поле + быстрые кнопки

Что **НЕ должно** быть в изображении. Поддерживается: Stable Diffusion (отдельное поле), Midjourney (через `--no`). Не поддерживается: DALL-E 3 и Flux.

**Быстрые чекбоксы-кнопки:**
- **Анти-брак:** `blurry, low quality, artifacts, noise`
- **Анти-CGI:** `cartoon, CGI, unrealistic`  
- **Анти-пластик:** `airbrushed, plastic skin, overly smooth`
- **Анти-анатомия:** `extra fingers, deformed hands, bad anatomy, mutated`
- **Анти-текст:** `text, watermark, signature, logo`
- **Анти-кроп:** `cropped, out of frame, cut off`

Кнопки добавляют текст к уже имеющемуся в поле (не заменяют его).

Финальный текстовый запрос, готовый к использованию.

## Часть 6. Пошаговый алгоритм — Ваш первый промпт

### За 30 секунд:
1. **Выберите движок** (раздел «AI Модель») — например, Midjourney
2. **Опишите сцену** (раздел «Описание сцены») — `tired detective in rainy neon city`
3. **Нажмите кнопку «Улучшить»** — ИИ добавит деталей
4. **Выберите крупность** (Параметры съёмки → Shot Size) — Medium Shot
5. **Выберите освещение** (Схемы света) — Neon Lighting
6. **Скопируйте промпт** (правая панель, кнопка «Копировать»)
7. **Вставьте в нейросеть** (Ctrl+V в Midjourney или другой системе)

### За полную профессиональную настройку:
1. Откройте Навигатор (кнопка «Конструктор ▶» в шапке)
2. Нажмите «Pro» — разблокируете все секции
3. Выберите движок → Опишите сцену → Выберите камеру → Объектив → Свет → Цвет → Атмосферу → Стиль
4. При необходимости загрузите референс
5. Выберите формат (Flat / Structured / Midjourney) в правой панели
6. Скопируйте результат
