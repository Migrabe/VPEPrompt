# Prompt Options Logic Validator

## Цель (Goal)
Глубоко проанализировать все данные, значения и функции на сайте генератора промптов для изображений. Выявить любые логические пересечения или конфликты между опциями (например, стили, освещение, типы рендера, параметры камеры) и создать полную, грамотную и бесконфликтную структуру параметров.

## Инструкции (Instructions)

1. **Сбор и анализ данных (Data Ingestion):** 
   - Изучи предоставленный пользователем код (HTML/JS/React), структуру данных (JSON/TypeScript интерфейсы) или схему базы данных, отвечающую за выбор опций генерации промпта.

2. **Картирование функций (Mapping & Deep Analysis):** 
   - Составь полную карту всех доступных значений. 
   - Определи функцию каждого значения: какой визуальный вес оно имеет и на какую часть итогового промпта (subject, medium, style, resolution, lighting, color) оно влияет.

3. **Поиск пересечений (Intersection & Conflict Analysis):**
   - Проверь каждую опцию на предмет логического дублирования или взаимоисключающего конфликта с другими опциями (например, выбор «Фотореализм» в одной категории и «Flat 2D вектор» в другой, или дублирующиеся параметры вроде «Cinematic Lighting» и «Studio Lighting», которые могут конфликтовать).
   - Выяви неявные зависимости (например, если выбрана опция «Пиксель-арт», опции «Объектив камеры» должны становиться недоступными).

4. **Реструктуризация (Restructuring):** 
   - На основе анализа собери новую, строгую структуру данных (например, древовидную JSON-схему или Pydantic/TypeScript модели).
   - Разбей опции на независимые, логически изолированные категории.
   - Пропиши логику условных зависимостей (if/then) для UI сайта, чтобы при выборе одной опции несовместимые с ней блокировались.

5. **Генерация отчета (Output):**
   - Выведи подробный Markdown-отчет с разделом «Найденные пересечения и логические ошибки».
   - Предоставь итоговую переработанную архитектуру опций в виде готового кода или JSON-конфигурации.

## Ограничения (Constraints)
- Не удаляй уникальные опции безвозвратно. Если опции пересекаются, создай для них правильную иерархию (например, подкатегории) или объедини их в один dropdown/select.
- Итоговая структура должна строго гарантировать, что пользователь физически не сможет собрать логически противоречивый промпт.
- Всегда подробно объясняй, почему конкретные опции были признаны пересекающимися.
